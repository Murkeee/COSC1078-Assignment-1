My project idea

Overview

The Overall idea of this project is to create a ‘smart mirror’ implementation which will:

a) Activate upon the detection of a user in front of the mirror (motion detection will likely be used)

b) Collect and display important daily information such as the weather, current news events or user chosen news topics.

c) Use the collected data to help the user determine the best course of action for skincare and clothing-choice for any given day.

Its also intended for the product to have user customizability and personalization options which can be controlled using the matching mobile application which allows the user to adjust and alter the location of widgets, etc.

Motivation

Although many people have access to so much information in the palm of their hands, not all these people have the time or patience to go through multiple sites and google searches just to find this information. I believe this product will help eliminate the hassle by collecting and displaying information for the user with recommendations on clothing choice and skincare choices based on the collected information. Moreover, with the recent event of Covid-19 and its ‘lockdowns’ there has been a surge in the popularity of the already rapidly growing selfcare and skincare markets meaning more people are realizing the importance of selfcare and skincare this is evident in Morder Intelligence’s market report labelled “Australia skin care products market – growth, trends, covid-19 impact, and forecasts (2022-2027)” This is important because while there is a growth in the market it means there are also more people who simply use skincare products without a second thought because it might’ve been recommended to them by another person rather than having recommendations given based on user input and data. Another reason for this project is to reduce the screen usage time when waking up as I know many struggle with always checking their phone in the morning, this project will help to reduce the screen usage time by providing another method of getting the information you need to go throughout your day.

Description

As described briefly in the overview, the product has 3 main functions:

FIRST

First, the product should activate when it detects a user in front of it. There are several ways to implement this function including the use of Video, infrared light, sounds, kinetic energy, etc. The method most suitable for this product is likely to be infrared light or Visible light. I will most likely use visible light such as a video camera as it not only provides the function of motion detection but can also provide important and more accurate data on the movement of the user and therefore is better future proofing as it provides an avenue for more features to be added later down the life of the product if needed. Using a camera will also make it possible to add features such as gesture control and can assist in the products function of providing advice on clothing choice and skincare choice by analyzing image data and processing it into sensible information which can be shown to the user.

SECOND

Second, the product will collect and display information on the weather and relevant news events. The collection of weather data is likely the simplest part as there are many ways to gather this data for free without the use of proprietary systems. One way weather information can be streamed is using weather API’s which are generally free to use, some examples include: OpenWeatherMap, Weatherbit, Accuweather and Tomorrow.io. Similarly, to weather APIs, there are also news APIs which can be used to collect and therefore display news to the user. These APIs are lightweight meaning they can be easily implemented in the product using something like Raspberry Pi and provide accurate weather forecasts, alerts, and conditions as well as the latest and up-to-date news for any given topic making them perfect for the product in question.

THRID

Third, the product should use and/or collect the appropriate data to provide the user with recommendations on daily clothing choice and skin-care choice. The former would be the easier of the two to implement as we can easily provide clothing suggestions based on weather information such as temperature, humidity, wind speed and precipitation which would have already been collected using the aforementioned weather APIs. Using the weather data, we can make suggestions such as wearing warmer clothes due to lower temperature or bringing an umbrella due to the precipitation predictions. For any given day. The latter function of providing skin-care advice is the more difficult of the two as while it is easy to say something like “put on sunscreen, it’s sunny” it’s very difficult to provide precise advice and information about the health of a person’s skin. One solution may be to use some AI to analyze an image and output information based on the AI’s analysis, while this is doable it would take a large amount of R&D to produce something tangibly accurate and precise enough to match the needs of each individual user as we would also have to take into consideration different skin-types and user needs.

Those are the three main functions of the product. Some auxiliary features that should be considered after the main functions have been implemented is user personalization and customization. Many people of our target audience value the aesthetics of products and often that can be a dealbreaker, so it is important that user personalization and customization are available whether that be using an external control such as a smart phone or a remote or through inbuilt controls like a side panel or touch screen type control. Voice control should also be considered as it is a relatively simple implementation in which we can use an AI API such as Google Assistant which will use a microphone input to catch voice commands.

Tools and technology

The required tools for prototyping would include simple carpentry tools such as drills, saws etc. We would also require access to a computer with which I will use VS code as that is my preferred software for coding. Hardware required would include a mirror, a frame and a monitor, as well as a Raspberry Pi on which the mirror will operate on. I will likely use glass on which I will attach one-way glass onto rather than using a fully opaque mirror. I have also thought about the possibility of water proofing using an ‘o-ring’ like seal around the frame to prevent moisture from entering the product and therefore prevent damage. For prototyping it could be desirable to have access to a 3D printer for more rapid prototyping.

Skills and talent required

The skills required include some hands-on skills with DIY projects of which is not difficult to find, and skills in the use of Raspberry Pi and several programming languages, which is also feasible. The most difficult part would be the ‘skin-care’ feature which if we are being ambitious would require personnel with experience in AI programming and image processing, which I believe is likely not as feasible and this function will likely not be implemented in the first iteration prototype. The hardware is the easiest part as all the required tools and technology are easily available online and in stores.

Project Outcome

If the project is successful, the product will improve daily lives by changing the way we wake up every day and instead of waking up to our phones we will wake up to the mirror and all the information you need is already there therefore reducing phone screen time in the morning. Overall, the project will help reduce the time it takes to get ready after waking up and will also benefit the user’s eye health and possibly even mental health by reducing the need to check one’s phone in the morning.
